Cold-start movie recommendation

Designed a practical approach to improve recommendation relevance for new users with no interaction history.

View GitHub →
Tools
Python · Embeddings · Similarity analysis
Data
Movie metadata (titles, genres, descriptions)
Focus
Cold-start logic + product relevance
Impact
Improved early-stage recommendation quality

Business problem

New users often receive poor recommendations because systems lack historical interaction data. The goal was to improve early recommendation relevance without relying on user behavior.

Data and constraints

  • Movie metadata including titles, genres, and descriptions.
  • Little-to-no interaction history for new users.
  • Need for a fast, deployable approach suitable for early-stage products.

Approach

  1. Generated semantic embeddings from movie content using pre-trained language models.
  2. Computed item-to-item similarity with cosine similarity.
  3. Designed logic that transitions from content-based recommendations to collaborative signals as data accrues.
  4. Benchmarked performance against simple popularity-based baselines.

Key insights

  • Embedding-based similarity significantly improved relevance for cold-start users.
  • Content-based recommendations outperformed popularity baselines for niche interests.
  • Simple similarity logic can be production-ready without heavy infrastructure.

Outcome

Delivered a clear, interpretable cold-start recommendation strategy with measurable improvements over baseline approaches, suitable for early-stage personalization.

What I’d do next

  • Incorporate lightweight feedback signals (likes, skips) to refine recommendations.
  • A/B test recommendation logic to quantify engagement lift.
  • Extend the approach to other domains such as articles or products.
