Phishing scam detection (applied NLP)

Identified high-signal, explainable risk indicators to support faster and more reliable security triage decisions.

Business problem

Security teams handle large volumes of suspicious messages and can’t manually review everything. They need a reliable, explainable way to prioritize cases for investigation—signals analysts can trust and audit.

Data and constraints

  • Unstructured text + metadata with inconsistent labeling.
  • Severe class imbalance (true phishing cases are rare).
  • Explainability requirement: analysts must understand why something is flagged.

Approach

  1. Normalized and cleaned text using a Python preprocessing pipeline.
  2. Explored simple, high-signal text indicators (urgency language, suspicious phrasing).
  3. Engineered interpretable metadata features (URL patterns, domain mismatch signals, attachment flags).
  4. Built Tableau views to expose drivers and support triage workflows.

Key insights

  • Urgency language + URL patterns captured a large share of high-risk cases.
  • Metadata features often provided stronger signal than complex embeddings.
  • Explainable indicators increased analyst trust and made the workflow usable in practice.

Outcome

Delivered an analyst-friendly prioritization approach using explainable risk indicators and reporting views. This improved triage speed while keeping full analyst oversight for final decisions.

What I’d do next

  • Add an analyst feedback loop to refine indicators over time.
  • Monitor shifts in phishing language and retrain/refresh features as patterns change.
  • Combine rule-based signals with small interpretable models for scalability.
