NLP · Security Triage

Phishing Scam Detection

Explainable risk indicators to prioritize suspicious messages faster — without removing human oversight.

Problem

Security teams receive high volumes of suspicious messages and can’t manually review everything. They need a reliable, auditable way to prioritize cases — signals analysts can trust and explain.

Approach

  • Cleaned and normalized unstructured text using a Python preprocessing pipeline.
  • Engineered interpretable indicators (urgency language, suspicious phrasing) and metadata features (URL patterns, domain mismatch, attachment flags).
  • Handled real-world data issues: class imbalance and label noise.
  • Delivered Tableau views to surface drivers and support an analyst triage workflow.

Insights

  • High-signal indicators: urgency language and URL patterns captured a large share of high-risk cases.
  • Metadata wins: simple metadata features often outperformed complex embeddings for actionable triage.
  • Trust matters: explainable flags made the workflow usable and auditable for analysts.

Outcome

  • Delivered an analyst-friendly prioritization approach using explainable risk indicators.
  • Improved triage speed while keeping human oversight for final decisions.

Next steps

  • Add an analyst feedback loop to refine indicators over time.
  • Monitor shifts in phishing language and refresh features as patterns change.
  • Combine rule-based signals with small interpretable models for scalability.
